# 双人合作攀岩 Demo

一个基于 Canvas 的双人合作攀岩原型。当前版本包含攀岩者四肢控制、攀岩机器人协作、物资拿取与使用、分段肢体耐久、软绳约束，以及内置地图制作工具。

## 运行

```bash
node server.js
```

默认地址：

```text
http://localhost:5177/
```

如果需要换端口：

```bash
$env:PORT=5180; node server.js
```

## 文件结构

```text
index.html   页面结构：游戏区、状态面板、联机设置、地图制作入口
styles.css   游戏和地图制作工具的样式
game.js      核心逻辑：输入、物理、IK、物资、地图编辑器、渲染
server.js    静态文件服务和轻量 WebSocket 房间同步
```

`game.js` 目前仍是单文件原型，内部按以下顺序组织：

1. DOM、配置和全局状态
2. 地图制作器与自定义地图载入
3. 游戏主循环、联机、机器人、绳子和攀岩姿态
4. 四肢 IK、物资、快照和输入控制
5. Canvas 渲染和 UI 更新

## 操作

### 攀岩者

- `Q / E / A / D`：选择左手、右手、左脚、右脚
- 鼠标移动：移动当前选中的肢体
- 鼠标点击：把当前肢体固定到当前位置
- `F`：靠近机器人时拿取物资；手持物资靠近目标时使用物资
- `Ctrl+C`：复制当前姿态快照数据到剪贴板和控制台
- `R`：重置游戏

### 攀岩机器人

- 方向键：移动机器人
- `Space`：固定或解除固定

机器人与攀岩者之间有一条软绳。绳长使用 `CFG.moveRadius`，视觉上是多段点模拟的下垂软绳，玩法上仍限制双方最大距离。

## 当前玩法系统

### 四肢与 IK

每条肢体都有根点、关节点和末端点。手臂和腿部通过二段 IK 计算姿态，接近最大伸展时会平滑趋近伸直，避免硬跳变。

身体在选中手尝试够远处时会产生“身体意图”，由 `updatePose()` 统一合并到身体目标中。这样可以表现从蹲姿站起、侧身、下探等动作，同时避免多个系统直接抢写身体位置。

### 肢体耐久

每条肢体拆成两段：

- 手：上臂、前臂
- 脚：大腿、小腿

每段有 `condition`：

- 健康：原色
- 亚健康：黄色
- 危险：红色

当前受力仍复用整条肢体的 `limb.load`，再按伸展程度分配到上下段。耐久下降会降低该肢体发力，影响最大可达距离和疲劳累积。

### 物资

机器人默认携带：

- 水
- 压缩饼干
- 绷带

水和压缩饼干需要送到嘴边后按 `F` 使用。绷带需要送到受伤肢体段附近后按 `F` 包扎，恢复该段耐久。

## 地图制作工具

顶部可切换“游戏 / 地图制作”。

地图制作模式提供：

- 可调整宽高的编辑区域
- 横向参考线，间距为人物最大伸展高度
- 底部素材栏
- 点击放置素材
- 保存并立即用于游戏

素材分为两类：

- 抓握类：小抓点、大突起、薄边、粗糙面
- 环境类：草药、小花、苔藓、裂纹

自定义地图保存在浏览器 `localStorage` 中，键名为：

```text
double-climb-custom-map
```

当前地图至少需要 4 个真正抓点类素材，小抓点、大突起或薄边，才能用于游戏初始四肢落点。粗糙面是可抓握区域，但不作为初始落点。

## 联机

`server.js` 提供一个非常轻量的 WebSocket 房间：

- 房主创建房间
- 加入者输入房间码
- 双方选择不同角色：攀岩者或攀岩机器人
- 房主负责模拟权威状态，并同步给加入者

该联机实现适合本地原型验证，不包含认证、断线重连、延迟补偿或生产级错误处理。

## 目前边界

- 地图制作器第一版只支持点击放置、保存和清空，还没有拖动物件、删除单个物件、撤销/重做。
- `game.js` 仍是单文件原型。继续扩大功能时，建议拆分为 `map-editor.js`、`physics.js`、`rendering.js`、`network.js` 等模块。
- 分段耐久是游戏化近似，不是真实刚体/肌肉受力模拟。
- 攀岩机器人在内部代码里仍有部分变量名沿用 `stone`，这是为了降低重构风险；用户可见文案已统一为攀岩机器人。
